# rpi4b-realtime-ml

## 1) Project Overview
This repository documents **four phases of CPU stress testing and temperature monitoring** on a **Raspberry Pi 4 Model B**. The goal is to understand how CPU load and core-affinity choices affect temperature, and to verify whether the Pi enters a throttled state under sustained load.

Temperature and throttling were checked using:
- `vcgencmd measure_temp`
- `vcgencmd get_throttled` (expected **`throttled=0x0`** for “no throttling”)

## 3) Phase-by-Phase Summary

### Phase 1 - Baseline / Core Visibility (setup)
**What was done:** Established baseline conditions (idle temperature and basic system info capture) and validated the environment needed for subsequent phases.

**Observation:** Idle temperature was approximately **~51°C** (environment-dependent).

---

### Phase 2 - All-core CPU stress (pure Python loop)
**What was done:** Ran a CPU-heavy loop for 30 seconds to generate load.

**Key code snippet** (`project-01-core-monitor/stress_test.py`):
```python
def cpu_run(seconds):
 end = time.time() + seconds
 while time.time() < end:
  _ = sum(i*i for i in range(10000))

print("Start running the CPU for 30s..")
cpu_run(30)
print("Done!")
```

**Observation:** CPU temperature rises noticeably under load. This phase is useful as a minimal-dependency stress test (no external packages), but it is not as intense/consistent as a dedicated stress utility.

---

### Phase 3 - Single-core pinning (CPU affinity)
**What was done:** Pinned the process to a specific core (core 2) and ran the same compute loop for 30 seconds. This demonstrates how **CPU affinity / core pinning** can control where work runs.

**Key code snippet** (`project-02-core-pinning/taskset_stress_test.py`):
```python
import os
os.sched_setaffinity(0,{2})
print("Pinned to core 2. Running for 30s.")
```

**Observation:** With pinning, the workload is concentrated on one core, which is useful for experiments involving real-time behavior, per-core thermal response, and scheduling effects.

---

### Phase 4 - Temperature logging under sustained load (stress + CSV)
**What was done:** Started a 4-core stress run for 60 seconds and logged temperature once per second to a CSV. After completion, throttling was checked via `vcgencmd get_throttled`.

**Key code snippet** (`project-03-temp-monitoring/temp_logger.py`):
```python
stress_process = subprocess.Popen(['stress', '--cpu', '4', '--timeout', '60'])

for seconds in range(60):
    result = subprocess.run(['vcgencmd', 'measure_temp'], capture_output=True, text=True)
    ...
result = subprocess.run(['vcgencmd', 'get_throttled'], capture_output=True, text=True)
```

**Observation:** From `project-03-temp-monitoring/temp_log.csv`, temperature ramped from about **~57°C** up to about **~69°C** during the captured interval. In general usage/testing, peak temperature was observed around **~75°C**, with **no throttling**.

---

## 4) Key Findings
- **Idle temperature:** ~**51°C** (approx.)
- **Peak temperature under load:** ~**75°C** (approx.)
- **Logged under sustained stress (sample run):** ~**57°C → ~69°C**
- **Throttling status:** `vcgencmd get_throttled` returned **`throttled=0x0`** → **no throttling detected**

> Note: Exact temperatures depend on ambient temperature and workload duration.

## 5) How to Run
### Prerequisites
- Raspberry Pi OS (or compatible Linux on Pi 4)
- Python 3
- `vcgencmd` available (typically preinstalled on Raspberry Pi OS)
- For Phase 4: install `stress`
  ```bash
  sudo apt-get update
  sudo apt-get install -y stress
  ```

### Phase 2: All-core (Python compute loop)
```bash
cd project-01-core-monitor
python3 stress_test.py
```

### Phase 3: Core pinning via Python affinity
```bash
cd project-02-core-pinning
python3 taskset_stress_test.py
```

### Phase 4: Stress + temperature CSV logging + throttling check
```bash
cd project-03-temp-monitoring
python3 temp_logger.py
```

Outputs:
- `temp_log_py.csv` (generated by the Python logger)
- `temp_log.csv` (example log captured via shell-style output format)
